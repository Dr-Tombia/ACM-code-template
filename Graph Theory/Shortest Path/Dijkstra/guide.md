# Dijkstra 算法 (单源最短路径) 核心解析

Dijkstra（迪杰斯特拉）算法是由荷兰计算机科学家 Edsger W. Dijkstra 发明的一种基于**贪心（Greedy）**策略的图论算法。它是解决**非负权图**单源最短路径问题的“绝对王者”，结合堆优化后，能将复杂度极大地压缩到 $O((V+E)\log V)$。

## 1. 使用前提条件

在使用 Dijkstra 算法之前，必须确认该题目满足以下两个核心条件：

* **核心禁忌：绝不能有负权边！** Dijkstra 的底层逻辑是“贪心”，它坚定地认为“只要我当前挑的是最短的路，以后继续走只会越来越远，不可能变近”。如果有负权边（越走距离反而越短），它的贪心策略就会彻底崩溃。（遇到负权边请左转使用 SPFA 或 Bellman-Ford 算法）。
* **单源最短路**：它只能计算从**一个固定的起点**出发，到达图中其他所有节点的最短距离。如果要求任意两点间的最短路，请使用 Floyd 算法，或跑 $n$ 次 Dijkstra。

## 2. 为什么普通的 BFS 会失败？

假设我们要从起点走到终点：
* 如果图中的边没有权重（或者权重都是 1），我们可以直接用普通的**广度优先搜索 (BFS)**，像水波一样一圈圈扩散，最先碰到的终点路径一定是最短的。
* 但如果是**带权图**（比如 A 到 B 距离是 100，A 到 C 距离是 1），普通的 BFS 按照“步数”扩散就会彻底失效，因为它只看跨过了几个节点，不看路有多长。

**Dijkstra 的核心思想**：带权重的“水波蔓延”。不以“经过的节点数”为基准，而是严格以“当前累积的最短物理距离”为基准进行扩散。

## 3. 核心灵魂：贪心与松弛 (Relaxation)

Dijkstra 算法的运作过程就是不断重复以下步骤，直到所有点都被确定：

* **初始化**：把所有节点到起点的距离（记为 `dist` 数组）全部设为无穷大 ($\infty$)，起点的距离设为 0。
* **找最小（贪心）**：在所有尚未确定最短距离的节点中，找出一个离起点**当前距离绝对最近**的节点 `u`。
* **敲定生死**：一旦节点 `u` 被挑出来，它的最短距离就被**永久敲定**了，以后无论如何绝不会再变短！将其标记为“已确定”。
* **松弛邻居（核心）**：把 `u` 当作跳板，看看它能直接走到的所有邻居 `v`。灵魂拷问：“从起点走到 `u`，再从 `u` 走到 `v`，是不是比目前已知的 `dist[v]` 更短？”如果是，就更新 `dist[v] = dist[u] + weight`。

## 4. 复杂度证明：为什么堆优化是 $O((V+E)\log V)$？

我们来看看找“距离最近的节点”这一步的代价（设 $V$ 为节点数，$E$ 为边数）：

**纯暴力流（朴素 Dijkstra）：**
每次找最小距离的节点，都要用一个 `for` 循环遍历所有的节点。找 $V$ 次最小节点，每次耗时 $O(V)$。加上更新边的耗时，总时间复杂度是 **$O(V^2)$**。在稠密图中表现不错，但在点多边少的稀疏图（如 $V=10^5$）中必然超时。

**优先队列/堆优化流（现代标配）：**
我们用一个**最小堆（优先队列）**来维护所有未确定节点的距离。
* 每次从堆顶直接“秒拿”距离最近的节点，耗时仅需 $O(\log V)$。
* 每条边最多被用来更新（松弛）一次，更新后将新状态压入堆中，单次压入耗时 $O(\log V)$。
* 两者结合，总时间复杂度被降维打击到了 **$O((V+E)\log V)$**！完美征服十万级别的大数据。

## 5. 实战避坑与代码细节

在编写 Dijkstra 模板时，有几个极易翻车的细节需要刻在 DNA 里：

1. **`INF` 设为 `1e9` 而不是 `INT_MAX`：** 在进行松弛操作 `dist[u] + weight` 时，如果 `dist[u]` 已经是 `INT_MAX`，加上一个正数会直接导致整型溢出变成负数，彻底毁掉答案。设为 `1e9`（即 $10^9$）既足够大表示无穷，又绝对安全。
2. **优先队列的玄学语法：** C++ 的 `priority_queue` 默认是大根堆。为了让距离最近的点在队头，必须写成 `priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>>`。并且 `pair` 的第一个元素**必须是距离**，第二个是节点编号。
3. **神来之笔：懒惰删除 (Lazy Deletion)：** 在取出队头元素时，必须加上一句 `if (d > dist[u]) continue;`。因为 C++ 的优先队列不能直接修改内部元素，同一个节点可能会带着不同的距离被多次压入堆中。如果取出的距离 $d$ 比最新记录的 `dist[u]` 还要大，说明这是个过时的废弃状态，必须直接跳过，否则会引发海量冗余计算导致 TLE。
